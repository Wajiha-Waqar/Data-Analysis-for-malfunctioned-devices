# Data Analysis for Malfunctioned Devices

## Project Overview

This project focuses on analyzing malfunctioned devices to identify patterns, trends, and operational issues using data visualization techniques. The analysis was performed using **Tableau**, enabling interactive exploration of device failures across different categories and metrics.

The objective is to help stakeholders understand which device types experience the most malfunctions, discover potential root causes, and support data-driven maintenance and operational decisions.

## Analysis Workflow

### 1. Data Collection

* Imported malfunction-related device data.
* Verified data quality and consistency.
* Removed duplicates and handled missing values where necessary.

### 2. Data Preparation

* Cleaned and structured the dataset.
* Categorized device types.
* Standardized fields for visualization.

### 3. Data Visualization

Interactive Tableau visualizations were developed to:

* Compare malfunction frequency by device type.
* Analyze failure distribution.
* Monitor performance indicators.
* Explore operational trends through filters and drill-down capabilities.

### 4. Dashboard Development

The dashboard was designed to provide:

* Clear KPI monitoring.
* Interactive filtering.
* Comparative device analysis.
* User-friendly navigation.
